Airtable Setup Guide — 4-Table Structure
This structure keeps one record per person and one record per job. Every time you interact with someone — first call, recruiter screen, third interview — it logs to Activity Log. No duplicates. Full history.
1
Table: Contacts — one record per person, ever
Name · Company · Role · How You Know Them · Notes (Long text) · Relationship Stage (Single select: Cold / Warm / Active / Advocate)
2
Table: Applications — one record per job, ever
Company · Role · Current Status (Single select: Researching / Applied / Phone Screen / Interview / Final Round / Offer / Rejected / Declined / Withdrawn) · Date Applied (Date) · Job URL (URL) · Job Description (Long text) · Notes (Long text)
3
Table: Activity Log — one record per interaction ← most important
Date (Date) · Activity Type (Single select: Networking Call / Recruiter Call / Company Call / Recruiter Screen / Interview / Offer / Rejected / Declined / Applied / Follow-up Sent / Other) · Contact ← Link to Contacts table · Notes (Long text) · Outcome
4
Table: Follow-ups
Due Date (Date) · Who ← Link to Contacts table · Why · Done (Checkbox)
5
Get your API credentials
Go to airtable.com/create/tokens → Create token with data.records:read and data.records:write scopes for your base. Copy the token + your Base ID from the URL (starts with app). Paste both in the Settings panel above.
