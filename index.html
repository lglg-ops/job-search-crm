Airtable Setup Guide — 4-Table Structure
This structure keeps one record per person and one record per job. Every interaction logs to Activity Log. No duplicates. Full history.
1
Table: Contacts — one record per person, ever
Name · Company · Role · How You Know Them · Notes · Relationship Stage (Cold/Warm/Active/Advocate) · Priority (High/Medium/Low) · Status · Type (Networking/Company/Recruiter) · Follow-up Tier (Active/Warm/Cold) · Snooze Until (Date)
2
Table: Applications — one record per job, ever
Company · Role · Current Status · Date Applied · Job URL · Job Description · Notes
3
Table: Activity Log — one record per interaction
Date · Activity Type · Contact ← Link to Contacts · Notes · Outcome
4
Table: Follow-ups
Due Date · Who ← Link to Contacts · Why · Done (Checkbox)
5
Get your API credentials
Go to airtable.com/create/tokens → Create token with data.records:read and data.records:write scopes. Paste token + Base ID (starts with app) in the Settings panel above.
